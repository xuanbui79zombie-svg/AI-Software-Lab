# Tasks

## 优先级

| 优先级 | 定义 | 响应原则 |
| --- | --- | --- |
| P0 | 阻断交付、安全事件、数据损坏或核心功能完全不可用。 | 立即处理，暂停低优先级工作。 |
| P1 | 影响主要用户流程或当前里程碑的重要任务。 | 当前周期优先完成。 |
| P2 | 有明确价值但不阻断交付的改进。 | 按计划排期。 |
| P3 | 探索、优化或低影响事项。 | 有余力时评估。 |

任务格式：`ID | 优先级 | 任务 | 验收标准 | 负责人 | 备注`

## 使用规则

- 新任务先进入 `TODO`，必须包含优先级和可验证的验收标准。
- 开始处理时移动到 `DOING`，避免同时进行过多任务。
- 完成并验证后移动到 `DONE`，记录完成日期。
- 无法继续时移动到 `BLOCKED`，说明原因、已尝试方案和解锁条件。
- 状态变化通过移动任务记录完成，不在多个区域复制同一任务。
