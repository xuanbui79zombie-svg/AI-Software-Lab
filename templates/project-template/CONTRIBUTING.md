# Contributing

感谢参与 `<PROJECT_NAME>`。提交贡献前，请阅读 `README.md`、`PROJECT_RULES.md`、`AGENTS.md` 和相关产品、架构文档。

## 流程

1. 搜索已有 Issue 和 Pull Request。
2. 用 Issue 说明问题、目标、范围、非目标和验收标准。
3. 从 `main` 创建短生命周期分支。
4. 实现聚焦改动并补充测试、文档和 Changelog。
5. 运行项目定义的格式、静态检查、测试、构建和安全检查。
6. 使用 Pull Request 模板提交证据、风险与回滚方式。
7. 解决评审意见并等待必需检查通过。

不得提交密钥、个人数据、生产数据、无许可证素材或无法验证的结果。安全问题必须按照 `SECURITY.md` 私密报告。

参与项目即表示同意遵守 `CODE_OF_CONDUCT.md`。
