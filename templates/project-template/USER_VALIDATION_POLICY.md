# Real-User Validation / 真实用户验证

## Current Status / 当前状态

`Not requested / 未要求`

Real-user validation is opt-in and off by default. Do not create recruitment, interview, usability-session, outreach, incentive, consent, or participant-data tasks unless the project owner explicitly requests validation for this project.

真实用户验证为选择加入，默认关闭。除非项目负责人明确要求本项目进行验证，否则不得创建招募、访谈、可用性会话、外联、激励、同意或参与者数据任务。

Engineering validation remains required: automated tests, code review, security, privacy, accessibility, compatibility, performance, deployment smoke tests, and documentation checks must still match project risk.

工程验证仍为必需：自动化测试、代码评审、安全、隐私、可访问性、兼容性、性能、部署冒烟测试和文档检查必须与项目风险匹配。

If validation is not requested, record `Not requested` or `Skipped by owner policy`. Never claim user validation, adoption, usability, satisfaction, market demand, product-market fit, or commercial outcomes without evidence.

未要求验证时，记录为 `Not requested` 或 `Skipped by owner policy`。没有证据时，不得声称已获得用户验证、采用率、可用性、满意度、市场需求、产品市场匹配或商业结果。
