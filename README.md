# AI Software Lab

AI Software Lab is the planning and management hub for my personal developer portfolio. It connects long-term career goals, project selection, technical growth, delivery quality, and resume-ready evidence in one place.

## Mission

My long-term goal is to become an independent developer capable of delivering commercial-quality software projects from problem discovery to deployment and iteration.

Read the complete mission and quality standards in [PORTFOLIO_MISSION.md](PORTFOLIO_MISSION.md).

## Portfolio Management System

| Document | Purpose |
| --- | --- |
| [PORTFOLIO_MISSION.md](PORTFOLIO_MISSION.md) | Defines portfolio goals, project selection standards, the technical growth path, quality requirements, and resume presentation standards. |
| [PROJECT_ROADMAP.md](PROJECT_ROADMAP.md) | Prioritizes future project ideas by technical and commercial value before development begins. |
| [PORTFOLIO_STATUS.md](PORTFOLIO_STATUS.md) | Tracks project repositories, technology stacks, completion, demos, and resume value. |

## Operating Workflow

1. Use the mission to decide what is worth building.
2. Evaluate and prioritize candidates in the roadmap.
3. Create a separate repository only after a project passes the selection criteria.
4. Track delivery progress and portfolio value in the status document.
5. Promote only well-documented, demonstrable work to the resume.

## Working Principles

- Start with a real problem and a measurable outcome.
- Keep projects focused, reproducible, and maintainable.
- Document decisions, trade-offs, tests, and results.
- Treat deployment, usability, and security as part of delivery.
- Share reusable knowledge with the open-source community.
