用于在开发前管理和评估未来项目。候选项目只有通过使命、价值和可交付性检查后，才创建独立仓库并进入开发。

## 优先级定义

| 优先级 | 含义 |
| --- | --- |
| P0 | 当前必须完成，直接影响求职、交付或核心能力证明。 |
| P1 | 高价值，P0 完成后优先启动。 |
| P2 | 有明确学习或展示价值，但不紧急。 |
| P3 | 观察项，价值或范围尚未验证。 |

## 技术价值评分

使用 1–5 分评估：

- 是否强化目标岗位需要的核心能力。
- 是否包含有意义的系统设计或工程挑战。
- 是否能沉淀可迁移、可复用的知识。
- 是否能证明测试、部署、安全和维护能力。
- 是否具有清晰的技术差异化。

## 商业价值评分

使用 1–5 分评估：

- 是否解决真实且明确的用户问题。
- 是否能够节省时间、降低成本、增加收入或改善体验。
- 是否存在可触达的目标用户和反馈渠道。
- 是否能够形成可演示、可验证的结果。
- 是否具备持续使用、迭代或商业化可能性。

## 项目准入条件

创建项目仓库前必须确认：

- [ ] 目标用户和核心问题明确。
- [ ] 最小可行版本范围可控。
- [ ] 成功指标和验收标准可验证。
- [ ] 技术价值和商业价值均已评分。
- [ ] Demo 和部署方式已有规划。
- [ ] 不涉及无法公开的敏感数据或授权风险。
- [ ] 与当前 Portfolio 目标和目标岗位相关。

## 管理流程

1. 收集项目想法，但不立即创建仓库。
2. 根据技术价值、商业价值和范围进行评分。
3. 设定优先级，并只保留少量可执行候选项目。
4. 通过准入条件后，创建独立项目仓库和验收计划。
5. 开发开始后，将项目加入 `PORTFOLIO_STATUS.md`。
6. 完成后复盘结果，并决定是否进入简历重点展示。
