# Portfolio Status

用于集中记录已进入开发或已完成的 Portfolio 项目。每次重要版本发布、Demo 更新或简历价值发生变化时同步更新。

| --- | --- | --- | ---: | --- | ---: |
| 暂无项目 | — | — | 0% | — | — |

## 记录规则

- **项目名称**：使用与项目仓库一致的正式名称。
- **GitHub 地址**：链接到项目的主仓库。
- **技术栈**：只记录实际使用并能够解释的核心技术。
- **完成度**：根据可交付成果评估，不以代码量代替进度。
- **Demo 地址**：优先填写公开可访问的在线 Demo；其次使用演示视频或截图。
- **简历价值评分**：使用 1–5 分制，并依据下方标准定期复评。

## 完成度定义

| 完成度 | 状态 | 判定标准 |
| ---: | --- | --- |
| 0% | 未开始 | 仅有想法，尚未通过项目选择标准。 |
| 25% | 已规划 | 需求、范围、架构和验收标准已明确。 |
| 50% | 开发中 | 核心流程已实现，但尚未达到可交付质量。 |
| 75% | 可演示 | 主要功能可用，已有测试和 Demo，仍需完善。 |
| 100% | 可展示 | 文档、测试、部署、演示和结果证据完整，可用于简历。 |

## 简历价值评分

| 评分 | 标准 |
| ---: | --- |
| 1 | 学习记录，缺少完整交付和岗位相关性。 |
| 2 | 功能可运行，但问题价值、质量或展示证据较弱。 |
| 3 | 完整可演示，能够证明至少一项目标岗位能力。 |
| 4 | 有真实价值、工程质量和可量化结果，可作为简历重点。 |
| 5 | 具备明显差异化、商业价值或开源影响力，是核心代表项目。 |

## 更新检查

- [ ] GitHub 地址有效且仓库可访问。
- [ ] 技术栈与实际代码一致。
- [ ] 完成度有对应的交付证据。
- [ ] Demo 可以正常访问或播放。
- [ ] 简历价值评分有明确依据。
