# Architecture Decision Records

Architecture Decision Records capture decisions that change multiple framework areas, constrain future repositories, introduce a dependency, or require a migration.

## Process

1. Copy `0000-template.md` to the next four-digit number and a short kebab-case title.
2. Set the status to `Proposed` and discuss material alternatives.
3. Change the status to `Accepted`, `Rejected`, or `Superseded` when decided.
4. Link a superseding ADR instead of rewriting an accepted decision.

Minor wording changes, routine maintenance, and decisions local to one business project do not need a framework ADR.
