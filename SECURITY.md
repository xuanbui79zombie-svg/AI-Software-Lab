# Security Policy

## Supported Versions

This repository is a documentation and automation framework rather than a deployed application. Security fixes are applied to the latest state of `main` and the latest published release, if one exists. Historical commits and superseded templates are not supported.

Projects created from this template must publish their own support window before production use.

## Report a Vulnerability

Do not open a public issue or discussion for a suspected vulnerability. Use [GitHub private vulnerability reporting](https://github.com/xuanbui79zombie-svg/AI-Software-Lab/security/advisories/new).

Include, when available:

- the affected file, workflow, template, or skill;
- impact and realistic attack scenario;
- reproduction steps or a minimal proof of concept;
- affected commit or version;
- suggested mitigation;
- whether the information has been shared elsewhere.

Never include real credentials, personal data, or destructive payloads in a report.

## Response Targets

The maintainer aims to acknowledge a complete report within 7 days, provide an initial assessment within 14 days, and coordinate disclosure after a fix or documented mitigation is ready. These are service targets, not guarantees.

## Disclosure and Credit

Please allow time for investigation and remediation before public disclosure. Valid reporters may receive credit in the advisory or release notes if they want it and if doing so is safe.

## Out of Scope

General support requests, unverified scanner output without an impact path, social engineering against maintainers, and vulnerabilities in unrelated downstream projects are out of scope. Use [SUPPORT.md](SUPPORT.md) for non-security help.
