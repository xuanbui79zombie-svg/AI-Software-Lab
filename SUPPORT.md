# Support

## Where to Ask

- Use the bug report form for a reproducible defect in this framework.
- Use the feature request form for a scoped improvement to governance, templates, automation, or Codex skills.
- Use a GitHub issue for public questions that could help future contributors.
- Follow [SECURITY.md](SECURITY.md) for suspected vulnerabilities; never publish them in an issue.

Before opening an issue, read the README and related source-of-truth document, search existing issues, and run `python scripts/validate_framework.py` when the problem involves repository structure.

## What to Include

Provide the goal, affected file or workflow, expected and actual behavior, reproduction steps, validation output, and relevant environment information. Remove credentials and personal data.

## Support Boundaries

This is a personal open-source portfolio framework maintained on a best-effort basis. It does not include guaranteed response times, private consulting, career placement, or support for downstream applications that have diverged from the template.

Questions about a separate project belong in that project's repository.
